About this document

1. The mirah language

It was previously called duby. It runs on the JVM and its syntax is Ruby.

  • Source file extension is .mirah
  • compiles mirah files using the mirah compiler (mirahc)
  • compiles to java byte code
  • can emit java source file if you pass the --java* in mirahc e.g. mirahc --java SourceFile.mirah
  • Easy to get started, if you already have an existing JDK installation and rubygems, just get mirah using gem install jruby mirah 
10# first.mirah
11# compile using: mirahc first.mirah
12# run using: java First
13
14puts "Hello mirah"
15System.out.println("hello java");

You can use the System.out of Java and the puts of ruby, mirah doesn't have a problem with it. The compiler emits First.class, it Uppercases the name of source file. It also does not impose enclosing your codes in a class--the emitted code will be enclosed in a class though, but don't bother with that. The point of mirah is so you can use Ruby syntax and run in a JVM

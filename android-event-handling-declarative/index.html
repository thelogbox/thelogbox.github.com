Simple Event Handling in Android (declarative)
by: Ted Hagos on 19.Jul.2012

Pre requisites for this tutorial

  1. Creating an android project
  2. Running a project

One of the two ways to handle events in Android programming is declaratively---which means we will declare the event using the appropriate xml file in the res folder. If you create a default android project (like how we have been creating android projects, there will be a main.xml inside the res folder. When you add views or widgets using that xml, you can declare the name of the function that will be called when an event is triggered, such as onClick.

Steps

  1. Create an android project named hellotoast (--path hellotoast)

    $ android create project --path hellotoast --package com.thelogbox --activity TestToast --target 8

  2. Add a Button view to the main.xml and set the method to call if the Button widget is clicked.

10<?xml version="1.0" encoding="utf-8"?>
11<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
12    android:orientation="vertical"
13    android:layout_width="fill_parent"
14    android:layout_height="fill_parent"
15    >
16<TextView
17    android:layout_width="fill_parent" 
18    android:layout_height="wrap_content" 
19    android:text="Hello World, TestToast"
20    />
21    
22<Button
23    android:layout_width="fill_parent"
24    android:layout_height="fill_parent"
25    android:text="Click me okay"
26    android:onClick="showMe"
27/>
28
29</LinearLayout>

  1. The android:onClick="showMe declaration of the Button widget means that when main.xml is expanded, it will create the necessary hooks for the proper handling of event. You will need to define a method called "showMe()" inside the TestToast activity of the project. The TestToast activity references the .main.xml as the View.

  2. Add the showMe() method inside /src/com/thelogbox/TestToast.java.

21import android.app.Activity;
22import android.os.Bundle;
23import android.widget.Toast;

26
27public class TestToast extends Activity {
28
29/** Called when the activity is first created. */
30
31    @Override
32    public void onCreate(Bundle savedInstanceState) {
33        super.onCreate(savedInstanceState);
34        setContentView(R.layout.main);
35    }
36
37    public void showMe(View v) {
38        Context ctx = getApplicationContext();
39        String msg = "Hello World";
40        int duration = Toast.LENGTH_SHORT;
41        Toast notification = Toast.makeText(ctx,msg,duration);
42        notification.show();
43    }
44
45}
  1. Compile and Run using $ ant debug install

What to remember

showMe() method - this is a programmer defined method. It could be any other name, as long as it is a valid Java identifier and a valid method.

When you declare an event inside the res/xml file, you need to ensure that a method with the same signature is found on the class that references the xml file as its ContentView---in this case, the TestToast class references main.xml as its content view, that is why the showMe() method is defined inside the TestToast class.

The Toast notification

Toast is a low hanging fruit for simple notifications. It is a built-in view that allows you display simple notifications to the user. These are the pop-up notifications you see on your android device. These notifications simply pops-up then disappears.

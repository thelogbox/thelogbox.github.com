Send and Receive SMS messages using a USB modem - Linux Guide
by: Ted Hagos on 9.Apr.2012

Objectives

  1. To receive and send (but mostly receive) SMS messages using consumer grade (meaning cheap) GSM modems.
  2. To do No.1 without resorting to WaveComm or iTegra type devices. The project I will be using it on does not require industrial grade throughput, hence, the price of a WaveComm modem is not justified
  3. To be able to query the SMS messages data-store either via file based parsing or database(d) queries -- database is preferred.

Possible solution approaches

  1. Write your own GSM modem interface. This requires writing the I/O routines. It's mostly serial programming . After that, just a grab a copy of the standard Hayes-AT GSM modem commands to receive/send SMS from the connected device. This should work for both Nokia and non-Nokia GSM modems
  2. Use Kannel as the SMS gateway. This is a very mature project. I'm pretty sure it is more than what you need.
  3. Use either gnokii-sms or gammu. These are also very mature projects like Kannel

The choice

No to writing your own software because I don't have a lot of time. The SMS component is small portion of the solution, this is not a core component. I cannot justify the amount of time that needs to be spent on research, development and testing.

No to Kannel because of the sheer size of the application. The learning curve that might be required could be sizeable (learning the APIs and how to integrate it with my application).

Yes to gnokii-sms. It is a stand-alone application with a daemon and it uses MySQL database as a datastore. I don't need to learn any API in order to integrate it with my application. All I need to do is to write an application that will query the gnokii-sms backend. I won't even alter the structure of the gnokii-sms database. My application will simply use gnokii for the SMS facility. No application logic will be added to gnokii-sms

What's needed

  1. Debian Squeeze for the platform. I could use Windows or any other Linux distribution but I am very familiar with Debian, so, Debian it is.
  2. Cheap USB modem with plan. I could have chosen pre-paid but I need to be mindful when the SIM is running out of pre-paid load credits.

How to do it

Get gnokii-smsd and mysql-server from the Debian repositories.

$ sudo apt-get install mysql-server
$ sudo apt-get install gnokii-smsd-mysql

Connect and configure the USB modem

  1. Connect the USB modem to Debian
  2. Find out which port it is using. On the Linux Terminal, issue either
$ sudo lsbusb 
$ sudo dmesg | grep USB

You are looking for the modem entry. The USB modem usually uses /dev/tty0, but just to be sure, run the commands anyway.
3. Configure gnokii-smsd to use the correct ports. The configuration file is at ~/.config/gnokii/config 4. You need to make 2 changes on the configuration file, the PORT and model entries. Set the port = /dev/tty0 (or whatever lsusb or dmesg has told you about the actual port). Set model = AT, the default is 6510 which assumes you are using a Nokia phone, you are not. You are using a non-nokia USB modem

NOTES

On some Debian variant (CrunchBang for example), the config file was not found in ~/.config folder, it does not exist at all. There was a config file though at /etc/xdg/gnokii/config. If this is case, you can simply create the configuration file and directory manually

$ mkdir ~/.config
$ cd ~/.config
$ mkdir gnokii
$ cd gnokii
$ cp /etc/xdg/gnokii/config config

Then make the edits as prescribed on Step No. 4 above.

To test if everything is in order, you can run the command

$ sudo gnokii --identify

This should give you some diagnostics and some quick peek if something is wrong with the configuration file

Prepare the database back-end for gnokii-smsd

I chose the MySQL variant of the gnokii-smsd, if you use another database like PostgreSQL, you can do that too, but you need to install gnokii-smsd-pgsql instead of gnokii-smsd-mysql.

Here is how to prepare the MySQL back-end

$ mysql -u root -p
$ mysql> create database smsgw;
$ mysql> use smsgw;
$ mysql>.\ /usr/share/doc/doc/gnokii-smsd-mysql/sms.tables.mysql.sql;

Line No. 4 above is the SQL script that you need to run in order to create the MySQL table structures which gnokii-smsd expects to find.

Running and testing the SMS gateway

To enable gnokii-sms to send and receive SMS messages, we need to run its daemon process.

$ sudo smsd -u root -p <MySQL Password> -d smsgw -c localhost -m mysql

Try sending some SMS messages to the phone number of the USB modem, then view the messages on the MySQL database.

$ mysql -u root -p
$ mysql> use smsgw;
$ mysql> select * from inbox;

Try sending some SMS messages from the command line to your phone.

$ echo "Test message" | gnokii --sendsmd +SMSNO

Where SMSNO is structured as (countrycode) cell phone number

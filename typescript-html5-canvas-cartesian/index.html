Line graphing in HTML5 Canvas
by: Ted Hagos on October 10, 2012

Motivations

  1. Graph lines in a cartesian plane
  2. Each vertex is associated with a line, vertices are named

Status of the experiment

Far from complete, but it is already fun to doodle with. The things that don't work yet are;

  • The lines cannot be repositioned yet. It doesn't work on FireFox and IE (the calls to event.clientX and event.clientY I think are the culprit here, I need to put some more thoughts on how to approach cross-browser functionalities).
  • The vertices are named up until "Z" only (although this is easy to fix, I just dont' want to give this time right now)
  • values cannot be inputted yet, then graphed. Needs more work

Approach

This is an exercise on the HTML5 canvas, so it will use canvas. Admittedly, this would (probably) be a lot simpler if this was done in SVG, because repositioning the vertices and the lines could be handled in a more straightforward way---each vertex and line is a DOM element, which means they can easily be bound to events such as click or mousedown. Anyway, it still a good coding calesthenics, so canvas it is.

The exercise has lots of parts. The cartesian plane needs to be drawn, a way to remember what has been drawn on canvas is also needed and the canvas needs a little bit of state management (counting the number of clicks)---one click means that we are about to draw a line, two clicks means we have chosen the second vertex and we need to do three things 1) complete the data of the line 2) store the information of the newly created line and then 3) redraw all of the previous lines we have created. It could be an ardous task if we won't try apply some form of abstractions on all of these things---a procedural-ish code might cut it, will cut it, but an OO style programming I think is warranted, even on this trivial exercise.

TypeScript came out just recently, even at its early stages, its OO and static type checking could be useful to this exercise, hence, TypeScript it is

The NOSCRIPT tag is also a quick and dirty way of detecting if JavaScript is disabled in the browser---I just don't see how this can kind of exercise can degrade gracefully if JavaScript is diabled, so its an all or nothing deal; No JavaScript, no demo.

2. The TypeScript file

234class Cartesian {
235    
236    private context;

239    public cartesian_unit: number = 40; // 40 pixels per cartesian unit
240    ax_tick_count: number;
241    ay_tick_count: number; 
242    
243    constructor(cx: any, c: any) {
244        this.context = cx;
245        this.canvas = c;
246        
247        this.ax_tick_count = Math.round(c.width / this.cartesian_unit);
248        this.ay_tick_count = Math.round(c.height / this.cartesian_unit);
249

251        this.drawY();

259    public drawX() : void {
260        
261        var cx : any = this.context;
262        var c : any  = this.canvas; 
263        var tickcount : number = c.width / this.cartesian_unit; 
264        var y_coord : number = this.cartesian_unit * Math.round(tickcount/2)//c.height /2;
265        var x_coord : number = 0;
266        var counter : number = Math.round(tickcount/2) * (-1);
267        
268        //cx.beginPath();
269        //cx.moveTo(0,c.height/2);
270        //cx.lineTo(c.width, c.height/2);
271        //cx.stroke();
272        
273        // draw the tick marks
274        
275        for(var i = 0; i <= tickcount; i++){
276            cx.beginPath();
277            if(counter == 0){
278                cx.moveTo(0,y_coord);
279                cx.lineTo(c.width,y_coord);
280            }
281            else {
282                cx.moveTo(x_coord, y_coord);
283                cx.lineTo(x_coord, y_coord + 5);
284            }
285            cx.closePath();
286            cx.stroke();
287            if(counter != 0) cx.fillText(counter, x_coord - 5, y_coord + 15);
288            counter++;
289            x_coord += this.cartesian_unit;
290        }
291    }

293    public drawY() : void {
294        
295        var cx : any = this.context;
296        var c  : any = this.canvas;
297        var tickcount : number = Math.round(c.height/this.cartesian_unit);
298        var x_coord : number =  this.cartesian_unit * Math.round(tickcount/2);//c.width/2;
299        var y_coord : number = 0; 
300        var counter : number = Math.round(tickcount/2) * -1;    
301                    
302        // draw the tick marks
303        
304        for(var i = 0; i <  tickcount; i++){    
305            cx.beginPath();
306            if(counter == 0){
307                cx.moveTo(x_coord,0);
308                cx.lineTo(x_coord,c.height);
309            }
310            else {
311                cx.moveTo(x_coord, y_coord);
312                cx.lineTo(x_coord+5, y_coord);
313            }
314            cx.closePath();
315            cx.stroke();
316            if(counter != 0) cx.fillText(counter * -1, x_coord - 15, y_coord + 5 );
317            counter++;
318            y_coord += this.cartesian_unit;
319        }

365        }
